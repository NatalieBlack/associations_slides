Rules

One-to-Many

  • foreign key on the "many side"

One-to-One

  • same as one to many
  • foreign key on one of the two sides

Many-to-Many

  • join table
  • i.e. a table with foreign keys for both things
  • could also have other columns

Identifying 1-Many Relationships in Your Data

  • Are there multiple columns that always have to change together?
  • If the value for this column changes in one row, will it also have to change in other rows?

  field :first_name, as: :string
  field :last_name,  as: :string

When should a join table have its own class?

  • Is there more information about this relationship?
  • Is there probably going to be more info about it in the future?
